Java设计模式：工厂方法模式

亦称： 虚拟构造函数、静态工厂方法、Virtual Constructor、Factory Method

工厂方法模式是一种创建型设计模式， 其在父类中提供一个创建对象的方法， 允许子类决定实例化对象的类型。

优点：

  • 将对象的创建与使用解耦，客户端只需要与工厂接口交互，无需关心具体实现。
  • 可以灵活添加新的产品类，符合开闭原则。

缺点：

  • 增加了系统的复杂性，需要额外的类和接口。
  • 如果产品类较多，工厂类可能会变得庞大。

适用场景：

  • 需要根据不同条件创建不同类型的对象。
  • 对象的创建逻辑复杂或需要灵活扩展。
