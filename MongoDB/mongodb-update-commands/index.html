更新单个文档的命令

以下是一些用于更新单个文档的 MongoDB 命令：

替换整个文档

  • update() 方法用于更新一个匹配的文档。
  • {"_id": 1} 是筛选条件，表示只更新 _id 字段为 1 的文档。
  • {"year": 2016} 是要替换的内容，它将完全替换匹配的文档。但需要注意的是，这会替换整个文档，包括文档中未指定的字段。

使用 $set 操作符更新部分字段

  • $set 操作符用于更新或添加文档中的字段。
  • {"_id": 1} 是筛选条件，表示只更新 _id 字段为 1 的文档。
  • {"year": 2016, name: "zhangsan"} 是要更新的内容，它只会更新或添加 year 和 name 字段，其他字段将不受影响。

使用 $unset 操作符删除字段

  • $unset 操作符用于删除文档中的字段。
  • {"_id": 1} 是筛选条件，表示只更新 _id 字段为 1 的文档。
  • {"year": 1} 表示要删除 year 字段。

使用 $rename 操作符重命名字段

  • $rename 操作符用于重命名文档中的字段。
  • {"_id": 1} 是筛选条件，表示只更新 _id 字段为 1 的文档。
  • {"year": "date"} 表示要将 year 字段重命名为 date 字段。

使用 $inc 操作符增加数字字段的值

  • $inc 操作符用于增加指定键的值。
  • {"_id": 1} 是筛选条件，表示只更新 _id 字段为 1 的文档。
  • {"year": 5} 表示要将 year 字段的值增加 5。

使用 $mul 操作符乘以数字字段的值

  • update() 方法用于更新一个匹配的文档。
  • {"_id": 1} 参数表示筛选条件，该命令将匹配 _id 字段值为 1 的文档。
  • $mul 操作符用于对指定键进行数值相乘。
  • price: NumberDecimal("1.25"), qty: 2 表示将 price 键的值乘以 1.25，将 qty 键的值乘以 2。

设置字段的最小值

以下是设置文档中字段的最小值的命令示例：

  • update() 方法用于更新一个匹配的文档。
  • {"_id": 1} 参数表示筛选条件，该命令将匹配 _id 字段值为 1 的文档。
  • $min 操作符用于将指定键的值设为文档中当前值和指定值中的较小值。
  • imdb: 5 表示将 imdb 键的值设置为 5，如果原来的值比 5 大，则保持不变。

设置字段的最大值

以下是设置文档中字段的最大值的命令示例：

  • update() 方法用于更新一个匹配的文档。
  • {"_id": 1} 参数表示筛选条件，该命令将匹配 _id 字段值为 1 的文档。
  • $max 操作符用于将指定键的值设为文档中当前值和指定值中的较大值。
  • imdb: 8 表示将 imdb 键的值设置为 8，如果原来的值比 8 小，则保持不变。

  • {"_id": 1} 参数表示筛选条件，该命令将匹配 _id

更新数组

将元素添加到数组中

  • update() 方法用于更新指定的文档。
  • {"_id": 1} 参数指定要更新的文档。
  • $push 操作符用于向数组中添加元素。
  • "array" 表示要更新的数组的键名。
  • 1 表示要添加的元素。

从数组中删除元素

  • update() 方法用于更新指定的文档。
  • {"_id": 1} 参数指定要更新的文档。
  • $pull 操作符用于从数组中删除指定的元素。
  • "array" 表示要更新的数组的键名。
  • 1 表示要删除的元素。

将元素添加到数组中（仅当元素不存在时）

  • update() 方法用于更新指定的文档。
  • {"_id": 1} 参数指定要更新的文档。
  • $addToSet 操作符用于向数组中添加元素，但仅当元素不存在时。
  • "array" 表示要更新的数组的键名。
  • 2 表示要添加的元素。

从数组中删除元素

  • update() 方法用于更新指定的文档。
  • {"_id": 1} 参数指定要更新的文档。
  • $pop 操作符用于从数组中删除指定的元素。
  • "array" 表示要更新的数组的键名。
  • 1 表示要删除的元素。

从数组中删除元素

  • update() 方法用于更新指定的文档。
  • {"_id": 1} 参数指定要更新的文档。
  • $pop 操作符用于从数组中删除指定的元素。
  • "array" 表示要更新的数组的键名。
  • -1 表示要删除的元素。

从数组中删除多个元素

  • update() 方法用于更新指定的文档。
  • {"_id": 1} 参数指定要更新的文档。
  • $pullAll 操作符用于从数组中删除多个元素。
  • "array" 表示要更新的数组的键名。
  • [3, 4, 5] 表示要删除的元素数组。

将多个元素添加到数组中

  • update() 方法用于更新一个匹配的文档。
  • {"_id": 1} 参数用于指定要更新的文档。
  • $push 操作符用于将一个或多个值添加到数组中。
  • scores 是要添加值的数组名。
  • $each 操作符用于指定要添加的多个值。

更新数组中的特定元素

  • updateOne() 方法用于更新一个匹配的文档。
  • {"_id": 1, "grades": 80} 参数用于指定要更新的文档和特定的 grades 数组元素。
  • $set 操作符用于更新指定键的值。
  • grades.$ 表示更新数组中第一个值为 80 的元素。

更新数组中的所有元素

  • updateMany() 方法用于更新多个匹配的文档。
  • {} 参数表示没有筛选条件，因此将更新所有文档。
  • $inc 操作符用于增加指定键的值。
  • grades.$[] 表示更新文档中的所有 grades 数组元素。

使用筛选条件更新数组中的元素

  • update() 方法用于更新一个匹配的文档。
  • {} 参数表示没有筛选条件，因此将更新所有文档。
  • $set 操作符用于更新指定键的值。
  • grades.$[element] 表示更新数组中满足筛选条件的元素。
  • multi: true 参数用于指定更新所有匹配的文档。
  • arrayFilters 选项用于指定筛选条件。此处的条件是更新数组中值大于等于 100 的元素。

更新很多

  • update() 方法用于更新匹配的第一个文档。
  • {"year": 1999} 参数表示要匹配的筛选条件。
  • $set 操作符用于设置指定键的值。
  • {"multi": true} 参数表示匹配到的所有文档都要被更新。
  • updateMany() 方法用于更新多个匹配的文档。
  • {"year": 1999} 参数表示要匹配的筛选条件。
  • $set 操作符用于设置指定键的值。

查找并更新

  • findOneAndUpdate() 方法用于查找一个文档并将其更新。
  • {"name": "zhangsan"} 表示查找 name 等于 "zhangsan" 的文档。
  • $inc 操作符用于增加指定键的值。
  • {"points": 5} 表示增加 points 的值。
  • {returnNewDocument: true} 参数表示返回更新后的文档。

更新插入

  • update() 方法用于更新匹配的文档，如果没有匹配的文档，则插入新的文档。
  • {"_id": 1} 表示查找 _id 等于 1 的文档。
  • $set 操作符用于设置指定键的值。
  • {"item": "apple"} 表示将 item 的值设置为 "apple"。
  • $setOnInsert 操作符用于设置插入时要设置的键和值。
  • {"defaultQty": 100} 表示插入文档时将 defaultQty 的值设置为 100。
  • {upsert: true} 参数表示如果没有匹配的文档，则插入新的文档。

替换

  • replaceOne() 方法用于替换一个文档。
  • {"name": "zhangsan"} 表示查找 name 等于 "zhangsan" 的文档。
  • {"firstname": "Maxime", "surname": "Beugnet"} 表示将文档替换为 {"firstname": "Maxime", "surname": "Beugnet"}。

保存

  • save() 方法用于保存一个文档。
  • {"item": "book", "qty": 40} 表示要保存的文档。

写关注（Write Concern）

MongoDB 允许您通过写关注（write concern）来控制写入操作的确认级别。以下是 update() 方法中的 writeConcern 参数的使用方法：

  • writeConcern 参数用于控制写入操作的确认级别。
  • "w": "majority" 选项告诉 MongoDB 确认写操作已成功写入到大多数节点。
  • "wtimeout": 5000 指定等待确认的时间（以毫秒为单位），在此时间内如果操作未被确认，则抛出错误。

注意：如果 writeConcern 参数未指定，则 MongoDB 将使用默认的写关注级别。默认情况下，MongoDB 会等待操作写入到 primary 节点后，立即返回成功响应。
